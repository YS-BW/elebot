# Agent Handoff

> 这份文件记录当前交接状态，只给接手 agent 使用。

## 当前任务背景

项目正在从旧的复杂方案中收口，目标是回到更清晰的 nanobot 风格终端助手基座。近期已经开始整理根目录 README 和 `docs/` 结构，但真正重要的是：

- 先稳住项目结构
- 再按代码规范清理代码
- 不要一边规范化一边偷偷扩功能

## 目前已经确定的事情

- 根目录 README 应该简洁，不承担详细设计说明。
- `docs/` 应该存放真实模块文档，而不是计划文档。
- Agent 交接文档不放进 `docs/`。
- 当前主线只实现旧系统已经有的功能，不补未来模块。

## 下一位 agent 最先要做什么

1. 确认本地执行环境可用，至少能运行基础命令和测试。
2. 阅读 `AGENTS.md`、`memory.md`、`takeover.md`、`docs/CODE_STYLE.md`。
3. 先核对代码结构是否符合当前文档描述。
4. 再对代码做规范化，不要先做功能重写。

## 当前不该做的事

- 不要重新设计一套新架构。
- 不要把 Frozen 模块接回默认主链路。
- 不要增加兼容层。
- 不要在看不到代码的情况下盲目大改。
- 不要把交接文档混进 `docs/`。

## 当前建议的推进顺序

1. 先清理和统一代码规范。
2. 再修正文档和测试的不同步问题。
3. 再做主链路的联调和行为校验。
4. 最后才考虑继续扩模块。

## 交接说明

如果接手 agent 发现：

- 文档和代码不一致
- shell 执行器不可用
- 多个 agent 同时改同一目录

应先停下来收口边界，再继续做实现。
