# Agent

Agent 模块负责一次用户请求从输入到最终回复的执行链路，是当前项目的核心模块。

## 当前职责

- 构造本轮对话上下文。
- 调用 Provider 获取模型输出。
- 处理流式正文与 thinking 内容。
- 识别并执行工具调用。
- 将工具结果回传模型，形成最终回答。
- 把本轮结果写入会话历史。

## 代码边界

- Agent 可以依赖 `config`、`providers`、`session`、`tools` 和 `bus`。
- Agent 不直接处理终端 UI。
- Agent 不读取用户输入，它只接收已经整理好的请求对象。

## 验收重点

- 单轮对话可完成。
- 多轮上下文可延续。
- 流式输出不是先全量收集再回放。
- 工具调用链路必须闭合。
