# 代码规范

这份规范是当前 `elebot` 仓库的最低基线，测试会按这里做静态校验。

## 文档字符串

- 每个 Python 模块都必须有模块级 docstring。
- 对外类、对外函数、核心方法必须有 docstring。
- docstring 必须使用中文。
- 参数和返回值复杂时，要在 docstring 里写清楚。

## 注释

- 注释优先解释“为什么”，不要重复解释“做了什么”。
- 普通注释默认使用中文。
- 分隔线注释可以不写中文，但不能代替正文说明。

## 结构

- 一个文件只负责一类清晰职责。
- `__init__.py` 只保留最小导出，不承载主逻辑。
- 不做无必要兼容层，不保留过渡别名。

## 文档
- 项目文档的目录结构为:
  - docs
    - elebot
      - `各个模块`
    - test
    - bridge
    - ....
    - 代码规范
    - ...
  - README.md
- `docs`下面必须存在每一模块的功能文档说明。
- 核心模块文档缺失时，代码改动不能视为完成。
- README 只能描述当前真实可用的默认主链路。

## 当前测试范围

当前自动测试重点检查：

- 文档文件是否存在
- Python 模块是否有中文模块 docstring
- 公共类/函数是否有中文 docstring
- 普通注释是否仍大量使用英文

注释是否真正解释“为什么”这一点，仍然需要人工 review。
